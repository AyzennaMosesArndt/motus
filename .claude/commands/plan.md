# /plan
Before implementing anything non-trivial, describe the approach and wait for approval.

## Format
TASK: what needs to be done
APPROACH: step-by-step plan (max 8 steps)
FILES AFFECTED: list of files to be created or modified
RISKS: what could go wrong
ESTIMATED COMPLEXITY: low / medium / high
WAITING FOR: human approval before proceeding

## When required
- Any change to pipeline stage contracts
- New DB migrations
- New agent or skill file
- Changes to settings.json permissions
